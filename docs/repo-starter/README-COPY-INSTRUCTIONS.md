# How to use this starter

These files are not meant to live in `roomwell-competitive-analysis` long-term. They are the bootstrap pack for the new private repo (`knutesteel/RoomwellAppClaudeCode`). Once the new repo exists, copy the contents of `docs/repo-starter/` to its root, then delete this folder from the analysis repo.

## Step-by-step

```bash
# 1. Confirm the new repo exists on GitHub (private)
#    Repo: knutesteel/RoomwellAppClaudeCode
#    Should be initialized empty (no README, no .gitignore — we have our own)

# 2. Clone the new empty repo somewhere local
git clone git@github.com:knutesteel/RoomwellAppClaudeCode.git
cd RoomwellAppClaudeCode

# 3. Copy the starter contents
#    Adjust paths to wherever you have the analysis repo checked out
cp -R ../roomwell-competitive-analysis/docs/repo-starter/. .
cp -R ../roomwell-competitive-analysis/docs/PRD.md docs/PRD.md
cp -R ../roomwell-competitive-analysis/docs/ARCHITECTURE.md docs/ARCHITECTURE.md
cp -R ../roomwell-competitive-analysis/docs/ROADMAP.md docs/ROADMAP.md
mkdir -p docs && mv PRD.md ARCHITECTURE.md ROADMAP.md docs/ 2>/dev/null || true

# 4. Verify hidden files came across (.gitignore, .github/, etc.)
ls -la
ls -la .github/workflows/

# 5. First commit
git add .
git commit -m "Initial monorepo scaffold"
git push origin main

# 6. From here, follow the README's "First-time setup" section to scaffold the
#    apps/web Next.js project, init Drizzle, etc.
```

## What's in the starter

| File | Purpose |
|---|---|
| `README.md` | Repo README — overview, setup, conventions |
| `.gitignore` | Standard ignores for Node + Next.js + Turborepo |
| `.env.example` | All env vars Roomwell needs, with comments |
| `package.json` | Root workspace manifest with shared scripts |
| `pnpm-workspace.yaml` | pnpm workspace declaration |
| `turbo.json` | Turborepo pipeline config |
| `tsconfig.base.json` | Shared strict TS config inherited by every package |
| `.editorconfig` | Editor consistency |
| `.prettierrc.json` + `.prettierignore` | Formatting |
| `.github/workflows/ci.yml` | CI pipeline (lint, typecheck, test, build, migration check) |

## What's NOT in the starter, and why

- **Next.js scaffold (`apps/web`)** — generated by `pnpm create next-app` per the README. Don't pre-bake what the upstream CLI generates better.
- **ESLint config** — the Next.js scaffold installs its own `eslint-config-next` baseline. We extend it from there.
- **Drizzle schema** — created by `pnpm dlx drizzle-kit` during initial setup. Schemas grow per milestone, not pre-baked.
- **Per-package `package.json` files** — generated via a `scripts/init-packages.ts` helper (documented in the README) once you decide which packages to bootstrap first.
- **Tailwind config** — comes from `create-next-app --tailwind`.

The principle: ship the scaffolding that the ecosystem CLIs don't, and let the ecosystem CLIs handle the rest.
